## 合同风险点分析

### 技术服务合同风险点
1. **费用支付风险**：分期支付，甲方可能拖欠尾款
2. **交付延期风险**：乙方可能延期交付，影响甲方项目进度
3. **知识产权风险**：乙方可能侵犯第三方知识产权
4. **保密义务**：保密期仅为3年，时间较短

### 保密协议风险点
1. **竞业限制补偿**：未约定补偿金具体金额，可能无效
2. **违约金数额**：100万可能过高，存在被法院调整的风险
3. **保密范围**：定义较宽泛，可能涵盖不应该保密的信息
4. **竞业限制期限**：2年可能超过合理范围

### 劳动合同风险点
1. **试用期工资**：试用期工资不得低于约定工资的80%（需确认是否符合）
2. **加班工资**：需确认实际加班费计算是否合法
3. **竞业限制**：未约定竞业限制补偿金
4. **解除合同**：甲方解除合同的条件是否合理

### 销售合同风险点
1. **预付款比例**：30%预付款比例较高，存在甲方资金风险
2. **交付期限**：30个工作日交货，时间较紧
3. **质保期**：36个月质保期较长，乙方承担风险较大
4. **违约金上限**：违约金设有5%上限，可能不足以弥补实际损失

## 生成 DOCX 和 PDF 文件

### 方法一：使用 Python（推荐）

安装依赖：
```bash
pip install python-docx PyPDF2 reportlab
```

运行生成脚本：
```bash
cd test-data/contracts
python generate_contracts.py
```

### 方法二：使用在线工具
1. 将 TXT 文件上传到在线转换工具（如 CloudConvert、ILovePDF）
2. 选择转换为 DOCX/PDF 格式
3. 下载生成的文件

### 方法三：使用 Microsoft Word / LibreOffice
1. 打开 TXT 文件
2. 另存为 DOCX 格式
3. 导出为 PDF 格式

## 合同类型标签

在上传合同时，应根据实际内容选择正确的合同类型：
- SERVICE：服务合同（如技术服务合同）
- SALES：销售合同
- NDA：保密协议
- EMPLOYMENT：劳动合同
- OTHER：其他类型

## 使用建议

1. **测试流程**：
   - 先上传 TXT 格式测试基本功能
   - 再测试 DOCX 和 PDF 格式，验证文件解析能力
   - 对比不同格式的分析结果

2. **风险验证**：
   - 检查系统是否能识别上述列出的风险点
   - 验证风险等级分类是否合理（高/中/低）
   - 确认证据链是否完整

3. **性能测试**：
   - 测试长文本（如技术服务合同）的处理性能
   - 测试批量上传多个合同的系统稳定性
   - 观察 Agent 处理各阶段的时间消耗
